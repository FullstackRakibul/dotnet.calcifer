# Security Policy for Calcifer

Security is a core foundation of the Calcifer microservice architecture. This template is designed for enterprise environments, and maintaining the integrity of its pre-configured Identity and JWT implementations is a top priority.

## Supported Versions

As Calcifer is currently in its active Alpha development phase, only the latest published pre-release version receives security updates. A formal long-term support (LTS) schedule will be established upon the official `1.0.0` stable release.

| Version         | Supported          |
| --------------- | ------------------ |
| 1.0.0-alpha.x   | :white_check_mark: |
| < 1.0.0-alpha.1 | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within this template—such as a flaw in the boilerplate authentication, role-based authorization configuration, or a vulnerable dependency—**please do not open a public GitHub issue.** Publicly disclosing a flaw could expose other developers using this template before a patch is available.

Instead, please report it privately:

1. **GitHub Security Advisories:** Navigate to the [Security tab](https://github.com/FullstackRakibul/dotnet.calcifer/security) of this repository and click "Report a vulnerability" to open a private advisory.
2. **Direct Contact:** If you cannot use GitHub Advisories, please reach out directly via professional channels (e.g., LinkedIn).

### What to Include in Your Report

To help triage the issue as quickly as possible, please include:

- A detailed description of the vulnerability and its potential impact on microservices generated by this template.
- Step-by-step instructions to reproduce the issue (a proof-of-concept repository or code snippet is highly appreciated).
- Any suggested mitigations or fixes.

## Response Timeline

All security vulnerability reports will be acknowledged within **48 hours**.

Once the vulnerability is verified, a patched version of the template will be pushed to the global NuGet registry, and a security advisory will be published. Reporters will be fully credited in the release notes for their responsible disclosure.
